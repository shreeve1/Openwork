# Agent Personas

A collection of ready-to-use AI instruction files for business, compliance, marketing, finance, education, real estate, technology, and other common work areas.

These files are meant to be uploaded into a Claude Project, ChatGPT project, custom GPT, Clawed web AI project, or another AI workspace that supports custom instructions or project knowledge.

Each `.txt` file contains instructions that tell the AI how to act like a specialized assistant. Many files include multiple sub-agents, menus, workflows, templates, and deliverables.

## What This Repository Is For

Use this repository when you want an AI assistant that is more focused than a normal chat.

Instead of asking a general AI a broad question like:

> Help me with marketing.

You can upload a specific instruction file such as:

> `marketing-sales/Marketing Strategist.txt`

Then the AI will behave more like a marketing strategy assistant with defined roles, workflows, and outputs.

These instruction files can help with:

- Business planning
- Marketing and sales
- Finance and accounting
- Legal and HR support
- Compliance guidance
- HIPAA and CMMC preparation
- Real estate workflows
- Restaurant and hospitality operations
- Teaching and classroom support
- Technology and IT support
- Non-profit operations
- Personal productivity
- Creative and design tasks

## Important Disclaimer

These files are AI instruction templates. They are not a replacement for licensed professionals.

For legal, tax, accounting, healthcare, cybersecurity, HIPAA, CMMC, or financial decisions, use these agents as planning and drafting helpers only. Always verify important advice with a qualified professional before acting on it.

Do not upload private passwords, API keys, Social Security numbers, protected health information, confidential customer data, or other sensitive information unless you fully understand the privacy rules of the AI platform you are using.

## What Is Inside Each Instruction File?

Most instruction files include some combination of:

- A system overview
- A startup message
- A menu of available agents or roles
- Activation instructions
- The AI persona or tone
- Responsibilities
- Step-by-step workflows
- Deliverables the AI should produce
- Questions the AI should ask the user
- Warnings, assumptions, or validation requirements

Some files are simple single-purpose assistants. Others are larger toolkits with several specialized agents inside one file.

## Basic How-To: Create an AI Project and Upload an Instruction File

The exact buttons will vary depending on the AI tool you use, but the general process is usually the same.

### Step 1: Choose the Right Instruction File

Browse the folders and pick the file that best matches what you want help with.

Examples:

- Need help with a business plan? Use `core-business-operations/Business Plan Assistant.txt`
- Need a marketing assistant? Use `marketing-sales/Marketing Strategist.txt`
- Need HIPAA compliance help? Use one of the files in `hipaa-compliance/`
- Need IT troubleshooting help? Use `technology-it/IT Support Troubleshooter.txt`
- Need classroom help? Use a file from `teachers/`

If you are not sure which file to use, start with the broadest file in the matching folder.

### Step 2: Create a New AI Project

In your AI tool, look for a feature called something like:

- Project
- Workspace
- Custom GPT
- Custom assistant
- Knowledge base
- Agent
- Bot
- Instructions

Create a new project and give it a clear name.

Examples:

- `Marketing Strategy Assistant`
- `HIPAA Compliance Helper`
- `Business Planning Assistant`
- `IT Support Troubleshooter`

### Step 3: Upload the Instruction File

Upload the `.txt` file into the project.

Depending on the platform, you may upload it as:

- Project knowledge
- A reference file
- Custom instructions
- System instructions
- Agent instructions

If the platform asks where to put the file, choose the option closest to **instructions** or **knowledge**.

### Step 4: Start a New Chat in That Project

After uploading the file, open a new chat inside the project.

Start with a simple message like:

> Please use the uploaded instruction file and show me the available menu.

Or:

> Please activate this assistant and ask me what information you need first.

Many of these files are designed to show a menu when the chat begins.

### Step 5: Follow the AI's Questions

The AI may ask questions before giving advice. Answer as clearly as you can.

For best results, include:

- Your goal
- Your business or organization type
- Your current problem
- Any deadline
- Any constraints
- What kind of output you want

Example:

> I run a small plumbing company with 8 employees. I need help creating a basic marketing plan for the next 90 days. My budget is $1,500/month and I mainly want more local service calls.

## How to Use These Files Without Uploading

If your AI tool does not support projects or file uploads, you can still use these instructions manually.

1. Open the `.txt` file.
2. Copy the full contents.
3. Paste it into a new AI chat.
4. Add this sentence at the top or bottom:

> Follow these instructions for the rest of this conversation.

This is less convenient than a project, but it usually works.

## Tips for Better Results

- Use one instruction file per project when possible.
- Start a new project for each major topic.
- Be specific about what you want the AI to produce.
- Ask for a checklist, plan, draft, table, or template if you need something concrete.
- Ask the AI to explain anything that feels confusing.
- Verify important facts before using the output.
- Keep sensitive information out of general-purpose AI tools unless your organization has approved that use.

## Adding New Instruction Files

To add a new persona or agent file:

1. Pick the best existing folder.
2. Create a new `.txt` file with a clear name.
3. Use a descriptive title at the top of the file.
4. Explain what the assistant is supposed to do.
5. Include startup instructions.
6. Include questions the assistant should ask.
7. Include the deliverables it should create.
8. Keep the file focused on one topic or one toolkit.

Suggested basic structure:

```markdown
# Agent Name

## Purpose
Explain what this assistant helps with.

## Startup Instructions
Tell the AI what to do when the file is loaded.

## User Questions
List the first questions the AI should ask.

## Responsibilities
List what the AI should help with.

## Deliverables
List the outputs the AI can create.

## Important Rules
List boundaries, disclaimers, or required behavior.
```

## Suggested Naming Style

Use clear names that basic users can understand.

Good examples:

- `Small Business Budget Planner.txt`
- `Restaurant Marketing Assistant.txt`
- `Employee Onboarding Guide.txt`
- `Cybersecurity Basics Advisor.txt`

Avoid unclear names like:

- `Agent 1.txt`
- `Helper.txt`
- `New File.txt`
