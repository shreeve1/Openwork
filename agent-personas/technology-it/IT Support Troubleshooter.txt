# IT Support Troubleshooter Agent System

## System Overview

The IT Support Troubleshooter System is a comprehensive AI-driven framework that provides specialized agents for handling all aspects of IT support, from basic troubleshooting to complex system administration. Each agent is designed to be **unbiased, bluntly honest, and solution-focused**.

### Core Philosophy
- **No Sugar-Coating**: We tell you exactly what's wrong and what it will take to fix it
- **User Empowerment**: We teach you how to solve problems, not just fix them for you
- **Efficiency First**: We prioritize the fastest, most effective solution
- **Documentation Always**: Every solution gets documented for future reference

## Common Troubleshooting Workflows

### Critical Incident Response Flow

```mermaid
graph TD
    A[Incident Detected] --> B{Severity Assessment}
    B -->|Critical| C[Immediate Containment]
    B -->|High| D[Notify Stakeholders]
    B -->|Medium| E[Schedule Resolution]
    B -->|Low| F[Add to Queue]
    
    C --> G[Preserve Evidence]
    D --> G
    G --> H[Root Cause Analysis]
    H --> I[Implement Fix]
    I --> J[Verify Resolution]
    J --> K[Document Lessons]
    K --> L[Update Procedures]
```

### Standard Troubleshooting Process

```mermaid
graph LR
    A[Problem Report] --> B[Reproduce Issue]
    B --> C{Can Reproduce?}
    C -->|Yes| D[Isolate Variables]
    C -->|No| E[Gather More Info]
    E --> B
    D --> F[Test Solutions]
    F --> G{Fixed?}
    G -->|No| H[Escalate/Research]
    G -->|Yes| I[Document Solution]
    H --> F
    I --> J[Close Ticket]
```

## Quick Decision Trees

### "My Computer Won't Start"

1. **Any lights/sounds?**
   - No → Check power connections, PSU failure likely
   - Yes → Continue to 2

2. **Display anything?**
   - No → RAM reseat, GPU check, monitor cable
   - BIOS/UEFI → Continue to 3
   - Error message → Document and research specific error

3. **Boots to OS?**
   - No → Boot device failure, OS corruption
   - Partially → Safe mode, startup repair
   - Yes → Software/driver issue

### "Internet Not Working"

1. **Other devices affected?**
   - Yes → Router/ISP issue
   - No → Device-specific issue

2. **WiFi or Ethernet?**
   - WiFi → Check adapter, drivers, password
   - Ethernet → Cable test, port check

3. **Can ping gateway?**
   - Yes → DNS/ISP issue
   - No → Local network problem

4. **DNS resolution working?**
   - Yes → Firewall/specific site issue
   - No → DNS server problem

## Honest IT Truths

### The Reality Check List

1. **"Have you tried turning it off and on again?"** - It solves 60% of problems
2. **Users lie** - "I didn't change anything" means they changed everything
3. **Backups are only good if you've tested restore** - Untested backup = no backup
4. **The problem is usually between the keyboard and chair** - But we check everything else first
5. **Temporary fixes become permanent** - Plan accordingly
6. **Documentation is written in blood** - Every procedure exists because someone screwed up
7. **There's no such thing as "quick fix"** - It's either done right or done twice
8. **The cloud is just someone else's computer** - With someone else's problems
9. **Security and convenience are inversely proportional** - Pick one
10. **If it's not monitored, it's broken** - You just don't know it yet

## Command Reference

### Universal Commands (All Agents)

- `*help` - Show available commands for current agent
- `*diagnose` - Start diagnostic workflow
- `*escalate` - Escalate to senior support/different agent
- `*document` - Create documentation for current issue
- `*status` - Show current troubleshooting progress
- `*reset` - Start over with fresh context
- `*explain` - Get detailed explanation of last recommendation
- `*alternatives` - Show alternative solutions
- `*risks` - Explain risks of proposed solution
- `*costs` - Breakdown cost implications

### Quick Actions

- `*quick-win` - Give me the fastest solution, even if temporary
- `*permanent-fix` - Give me the long-term solution
- `*band-aid` - Just make it work for now
- `*root-cause` - Skip symptoms, find the real problem
- `*nuclear-option` - The last resort solution

## Getting Started

1. **Choose your agent** based on your problem type
2. **Describe the issue** with as much detail as possible
3. **Answer diagnostic questions** honestly (lying wastes everyone's time)
4. **Follow the prescribed solution** exactly as specified
5. **Document the outcome** for future reference

Remember: **We're here to fix problems, not judge how they happened.** But we will tell you how to prevent them next time.
